# 📝 PRD Structure — 12 Mục Chuẩn

## Cấu Trúc

| # | Mục | Nội dung |
|---|-----|---------|
| 1 | Problem Statement | Vấn đề + Giải pháp + Business Value + KPIs |
| 2 | User Personas | Role, Mục tiêu, Pain points, Tech savvy |
| 3 | Scope | In-scope / Out-of-scope / Roadmap |
| 4 | User Stories + AC | As a / I want / So that + Given-When-Then |
| 5 | Functional Requirements | FR-001... with Priority |
| 6 | Non-Functional Requirements | Performance, Security, Availability |
| 7 | Data Model | Entities, Relationships, ER Diagram |
| 8 | UI/UX Wireframes | Layout, Components, Behavior |
| 9 | Technical Considerations | Dependencies, Constraints, Integrations |
| 10 | Timeline & Milestones | Deadlines + Deliverables |
| 11 | Risks & Mitigations | Probability × Impact |
| 12 | Open Questions | Câu hỏi cần trả lời |

## Checklist Viết PRD

- [ ] Mọi requirement đều SMART (Specific, Measurable, Achievable, Relevant, Time-bound)
- [ ] Mọi User Story đều có ≥ 3 Acceptance Criteria
- [ ] Có Error Scenarios cho mọi form/input
- [ ] Out of Scope rõ ràng kèm lý do defer
- [ ] Non-functional có metrics cụ thể (< 500ms, 99.9%)
- [ ] Version + Status ghi rõ (Draft / Review / Approved)
