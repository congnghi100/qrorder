# 🔎 Gap Analysis — Kỹ Thuật Phân Tích AS-IS / TO-BE

## 1. Framework Phân Tích

| Bước | Hoạt động | Output |
|------|----------|--------|
| 1 | Khảo sát AS-IS | Mô tả chi tiết hiện trạng |
| 2 | Xác định TO-BE | Mô tả mong muốn |
| 3 | So sánh Gap | Bảng AS-IS vs TO-BE |
| 4 | Phân loại Gap | Process / Technology / People / Data |
| 5 | Đề xuất giải pháp | Roadmap từng gap |

## 2. Phân Loại Gap

| Loại | Mô tả | Ví dụ |
|------|-------|-------|
| Process | Quy trình thiếu/sai | Không có approval workflow |
| Technology | Công cụ/hệ thống thiếu | Thiếu CRM, dùng Excel |
| People | Thiếu kỹ năng/nhân sự | Chưa có QA |
| Data | Dữ liệu thiếu/sai/rời rạc | Data trùng lặp, không tập trung |

## 3. Gap Priority Matrix

| Impact \ Effort | Low Effort | High Effort |
|----------------|-----------|------------|
| High Impact | 🔴 DO FIRST | 🟡 PLAN CAREFULLY |
| Low Impact | 🟢 QUICK WIN | ⚪ DO LATER |

## 4. Checklist Khi Phân Tích

- [ ] Liệt kê đầy đủ các khía cạnh: Process, Tech, People, Data
- [ ] Mỗi gap đều có: Mô tả, Impact, Priority, Owner, Timeline
- [ ] Gap có thể trace ngược về Business Objective
- [ ] Có roadmap rõ ràng (Phase 1, 2, 3...)
